# Store Performance Scoring Model

**Live demo:** [https://malatown-score-tool.vercel.app](https://malatown-score-tool.vercel.app)  
**Recommended source status:** Private source code

## Overview

Store Performance Scoring Model is an internal operations tool for evaluating store-level performance through structured data management and scoring workflows.

The project is designed to help operations teams move from scattered spreadsheet review toward a more standardized scoring model for store performance, follow-up prioritization, and management review.

## Key Features

- Store-level performance scoring workflow.
- Structured data entry and review experience.
- Spreadsheet-based data import support.
- Supabase-backed data storage.
- Operational score management for internal review.
- Dashboard-style interface for comparing store performance indicators.
- Deployable web application hosted on Vercel.

## Tech Stack

## My Role

## Business Value

This tool helps standardize store performance review by turning operational inputs into a more structured evaluation process. It supports more consistent management discussions and helps identify stores that may need follow-up, coaching, or deeper analysis.

Operational impact includes:

- More consistent performance review criteria.
- Faster preparation for store-level management review.
- Better visibility into underperforming or high-risk locations.
- A reusable foundation for future operations scoring and benchmarking.

## Privacy And Security Notes

This project should be treated as a private source-code repository. Public viewers should use the live demo and portfolio summary instead of source access.

This README intentionally avoids exposing:

- private scoring formulas if considered confidential
- store-level sensitive data
- API keys
- environment variables
- credentials
- internal business data

## Safe Demo Strategy

If the scoring logic or underlying operating data is sensitive, the public demo should use mock stores, mock scores, or a limited preview mode. Production data should not be used in public-facing demos.
